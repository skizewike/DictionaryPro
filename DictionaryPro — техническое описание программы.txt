DictionaryPro — техническое описание программы
Общее описание
DictionaryPro — это десктопное приложение для изучения иностранных слов с использованием метода интервальных повторений. Программа позволяет создавать уроки, добавлять слова, отслеживать прогресс и проходить тесты. Приложение работает локально, все данные хранятся на компьютере пользователя.

Архитектура программы
Программа построена на следующих технологиях

Backend Django (Python) — веб-фреймворк для обработки логики

Frontend HTML, CSS, JavaScript — интерфейс пользователя

База данных SQLite — легковесная реляционная база данных

Десктоп-оболочка Electron — превращает веб-приложение в нативное окно

База данных и нормализация (3NF)
База данных спроектирована в третьей нормальной форме (3NF). Это означает, что данные не дублируются, каждая таблица описывает одну сущность, а связи между таблицами осуществляются через внешние ключи.

Структура базы данных
Таблица Language (Языки)
Хранит информацию о языках, которые можно изучать.
Поля id (первичный ключ), name (название языка), code (код языка, например en, de, fr)

Таблица Category (Категории)
Хранит тематические категории слов.
Поля id (первичный ключ), name (название категории, например Еда, Животные, Одежда)

Таблица Lesson (Уроки)
Хранит информацию об уроках.
Поля id (первичный ключ), name (название урока), description (описание), language_id (внешний ключ к таблице Language), order (порядок уроков), is_active (активен ли урок)

Таблица Word (Слова)
Хранит слова для изучения.
Поля id (первичный ключ), term (иностранное слово), translation (перевод), language_id (внешний ключ к Language), lesson_id (внешний ключ к Lesson), created_at (дата создания)

Таблица Word_Categories (связь слов и категорий)
Промежуточная таблица для связи многие-ко-многим между Word и Category.
Поля id, word_id (внешний ключ к Word), category_id (внешний ключ к Category)

Таблица WordProgress (Прогресс слов)
Хранит прогресс изучения каждого слова для каждого пользователя.
Поля id, user_id (внешний ключ к User), word_id (внешний ключ к Word), mastered (изучено ли слово), correct_count (количество правильных ответов), wrong_count (количество неправильных ответов), last_answered (дата последнего ответа), next_review_date (дата следующего повторения), review_interval_days (интервал повторения в днях), review_stage (стадия изучения от 0 до 5)

Таблица CompletedTest (Пройденные тесты)
Хранит историю пройденных уроков-тестов.
Поля id, user_id (внешний ключ к User), lesson_id (внешний ключ к Lesson), score_percent (результат в процентах), correct_count (количество правильных ответов), total_count (общее количество слов), completed_at (дата и время прохождения)

Связи между таблицами
Language (один) → (много) Lesson (один язык может содержать много уроков)

Language (один) → (много) Word (один язык может содержать много слов)

Lesson (один) → (много) Word (один урок может содержать много слов)

Word (один) → (много) WordProgress (одно слово может иметь прогресс у многих пользователей)

User (один) → (много) WordProgress (один пользователь может иметь прогресс по многим словам)

User (один) → (много) CompletedTest (один пользователь может пройти много тестов)

Lesson (один) → (много) CompletedTest (один урок может быть пройден много раз разными пользователями)

Word (много) ←→ (много) Category (связь многие-ко-многим через промежуточную таблицу)

Система интервальных повторений
Программа использует модифицированную систему интервальных повторений. Каждое слово проходит 6 стадий изучения

Стадия 0 Новое слово (еще не отвечено ни разу)

Стадия 1 Начало (1 правильный ответ, повторение через 1 день)

Стадия 2 В процессе (2 правильных ответа, повторение через 3 дня)

Стадия 3 Укрепление (3 правильных ответа, повторение через 7 дней)

Стадия 4 Закрепление (4 правильных ответа, повторение через 14 дней)

Стадия 5 Мастер (5 и более правильных ответов, слово считается изученным, повторение через 30 дней)

При неправильном ответе стадия понижается на один уровень, а счетчик правильных ответов сбрасывается.

Безопасность и хранение данных
Все данные хранятся локально в файле db.sqlite3

Пароли пользователей хэшируются с использованием алгоритмов Django

Никакие данные не отправляются в интернет

Приложение работает полностью автономно

Системные требования
Операционная система Windows 10 или 11

Процессор 1 ГГц или выше

Оперативная память 4 ГБ (рекомендуется)

Свободное место на диске 500 МБ

Дополнительно Python 3.8 или выше, Django 4.0 или выше

Функциональные возможности
Регистрация и авторизация пользователей

Создание языков, уроков, категорий и слов через админ-панель

Изучение слов в режиме тестирования

Автоматический расчет интервалов повторения

Отслеживание прогресса по каждому слову и уроку

Статистика и история пройденных тестов

Личный кабинет с историей активности

Фильтрация слов в словаре по языку, категории, уроку и уровню изучения
